      JQuery est une librairie pour le langage Javascript. Souvent mentionnée à tort en tant que Framework Javascript, elle ne dispose pas d'une architecture, de modèle MVC, ou encore d'application interne au DOM comme peut le faire Angular JS.

      JQuery n'est rien d'autre qu'un ensemble de fonctions Javascript conçues pour faciliter l'écriture de ce langage. C'est ainsi qu'un document.getElementById('monId').style.display = 'none' en Javascript sera traduit par un $('#monId').hide() en JQuery.

      Le but de JQuery est la compréhension et la facilité du Javascript avant toute chose ainsi qu'une exploitation du langage.

      Beaucoup apprenent à programmer en JQuery avant de programmer en Javascript "pur" pour un apprentissage plus rapide du langage (Le Javascript étant un langage assez compliqué à comprendre au départ).

      On peut dire que JQuery est devenu un incontournable dans le monde du Web aujourd'hui, et est bien plus qu'une simple librairie aux yeux de beaucoup de développeurs (ce qui peut entraîner la méprise mentionnée ci-dessus).

      Personnellement, j'utilise aujourd'hui JQuery sur quasiment tous mes projets. Sa simplicité et son large ecosystème de fonctions sont un réel gain de temps, de lignes, et surtout de compréhension dans le code.

      Note : La faiblesse de JQuery, s'il faut en trouver une, est qu'il n'est utilisable qu'une fois la librairie chargée. Ce qui, lors de la création d'un widget destiné à être exporté sur des sites sans savoir si JQuery se trouve sur le site tiers, est embêtant.

      Angular JS est l'un des premiers frameworks Javascript qui s'immisce véritablement dans le DOM. Présent dans les balises HTML en tant qu'attribut (dans le sens ou un "ng-model" sera notifié dans une balise de la même façon qu'un attribut "rel" ou encore "src"), le grand point fort du framework est son injection de dépendances.

      Une variable appellée le scope navigue entre le(s) controller(s) de l'application et la vue (le HTML). Ainsi, si la variable est modifiée, son affichage le sera également et en direct, sans nécessité de rechargement ou attente tierce de la page. On appelle cela le "two way data binding", ce qu'on peut comprendre par une "actualisation en temps réel des données".

      En plus des attributs natifs d'Angular que l'on place dans les balises, on affiche les variables du scope directement dans le HTML, sous la forme {{$scope.pseudo}}. Cette formule est directement interprétée et affichée sous forme de texte (ou nombre) dans le HTML.

    • Node.js
      Liens utiles
      • Download
      • Documentation
      • NPM/

      La première technologie impliquant le Javascript au niveau du serveur. La dextérité du Javascript pour gérer le Back-end d'une application web. Node.js a fait beaucoup de bruits lors de son arrivée et beaucoup de développeurs (dont moi) en sont tombés amoureux.

      Node.js est livré avec NPM, un gestionnaire de packages (plugins) qui lui est dédié. C'est simple, on y retrouve tous les modules connus (des fonticons aux modules de templating, des packages les plus fous aux plus utiles).

      Les packages les plus connus sont surement Express qui est un framework permettant d'écrire plus simplement du code serveur Node.js, socket.io qui à lui seul mériterait un article et qui a pour concept l'utilisation du temps réel côté serveur, et enfin, EJS / Underscore qui sont des modules de templating bien pratiques pour afficher les variables données par le serveur Node sur le front.
